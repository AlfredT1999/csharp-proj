RULES WITHIN A NAMESPACE

# NAME SCOPING

NAMES DECLARED IN OUTER NAMESPACES CAN BE USED UNQUALIFIED WITHIN INNER NAMESPACES. IN THIS EXAMPLE, Class1 DOES NOT NEED QUALIFICATION WITHIN Inner:

namespace Outer
{{squareBraketOpen}}
   class Class1 {{squareBraketOpen}}}

   namespace Inner
   {{squareBraketOpen}}
      class Class2 : Class1 {{squareBraketOpen}}}
   }
}

IF YOU WANT TO REFER TO A TYPE IN A DIFFERENT BRANCH OF YOUR NAMESPACE HIERARCHY, YOU CAN USE A PARTIALLY QUALIFIED NAME:

namespace MyTradingCompany
{{squareBraketOpen}}
   namespace Common
   {{squareBraketOpen}}
      class ReportBase {{squareBraketOpen}}}
   }

   namespace ManagementReporting
   {{squareBraketOpen}}
      class SalesReport : Common.ReportBase {{squareBraketOpen}}}
   }
}

# NAME HIDING

IF THE SAME TYPE APPEARS IN BOTH AN INNER AND AN OUTER NAMESPACE, THE INNER NAME WINS. TO REFER TO THE TYPE IN THE OUTER NAMESPACE, YOU MUST QUALIFY ITS NAME:

namespace Outer
{{squareBraketOpen}}
   class Foo {{squareBraketOpen}}}

   namespace Inner
   {{squareBraketOpen}}
      class Foo {{squareBraketOpen}}}
      class Test
      {{squareBraketOpen}}
            Foo f1; // = Outer.Inner.Foo
            Outer.Foo f2; // = Outer.Foo
      }
   }
}

# REPEATED NAMESPACES

YOU CAN REPEAT A NAMESPACE DECLARATION, AS LONG AS THE TYPE NAMES WITHIN THE NAMESPACES DON'T CONFLICT:

namespace Outer.Middle.Inner
{{squareBraketOpen}}
   class Class1 {{squareBraketOpen}}}
}

namespace Outer.Middle.Inner
{{squareBraketOpen}}
   class Class2 {{squareBraketOpen}}}
}

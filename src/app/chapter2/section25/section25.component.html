# NAMESPACES

- A namespace IS A DOMAIN FOR TYPE NAMES. TYPES ARE TYPICALLY ORGANIZED INTO HIERARCHICAL namespaces, MAKING THEM EASIER TO FIND AND PREVENTING CONFLICTS.
- namespaces ARE INDEPENDENT OF ASSEMBLIES, WHICH ARE .dll FILES THAT SERVE AS UNITS OF DEPLOYMENTS.
- namespaces ALSO HAVE NO IMPACT ON MEMBER VISIBILITY -PUBLIC, INTERNAL, PRIVATE AND SO ON.
- THE namespace KEYWORD DEFINES A namespace FOR TYPES WITHIN A THAT BLOCK; FOR EXAMPLE:

namespace Outer.Middle.Inner
{{squareBraketOpen}}
  class class Class1 {{squareBraketOpen}}}
  class class Class2 {{squareBraketOpen}}}
{{squareBraketClose}}

THE DOTS IN THE namespace INDICATE A HIERARCHY OF NESTED namespaces. THIS IS SEMANTICALLY IDENTICAL TO THE PRECIDING CODE:

namespace Outer
{{squareBraketOpen}}
  namespace Middle
  {{squareBraketOpen}}
    namespace Inner
    {{squareBraketOpen}}
     class Class1 {{squareBraketOpen}}}
     class Class2 {{squareBraketOpen}}}
    {{squareBraketClose}}
  {{squareBraketClose}}
{{squareBraketClose}}

YOU CAN REFER TO A TYPE WITH ITS FULLY QUALIFIED NAME, WHICH INCLUDES ALL namespaces FROM THE OUTERMOST TO THE INNERMOST. TYPES NOT DEFINED IN ANY namespace ARE SAID TO RESIDE IN THE GLOBAL namespace.

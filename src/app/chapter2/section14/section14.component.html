PASSING ARGUMENTS BY VALUE

ARGUMENTS IN C# ARE PASSED BY VALUE. THIS MEANS THAT A COPY OF THE VALUE IS CREATED WHEN PASSED TO THE METHOD:

// Passing by value:
int x = 8;

Foo(x);// Make a copy of x

Console.WriteLine(x); // x will still be 8

static void Foo(int p)
{{ squareBraketOpen }}
  p = p + 1; // Increment p by 1
  Console.WriteLine(p); // Write p to screen
}

// Output :
// 9
// 8

- ASSIGNING p A NEW VALUE DOES NOT CHANGE THE CONTENTS OF x, BECAUSE p AND x RESIDE IN DIFFERENT MEMORY LOCATIONS.


PASSING A REFERENCE-TYPE ARGUMENT BY VALUE COPIES THE REFERENCE BUT NOT THE OBJECT:

// Pasing a reference argument by value:
using System.Text;

StringBuilder sb = new StringBuilder();

Foo(sb);

Console.WriteLine(sb.ToString()); // test

static void Foo(StringBuilder fooSB)
{{ squareBraketOpen }}
  fooSB.Append("test");
  fooSB = null; // JUST MODIFIES THE REFERENCE BUT NOT THE VALUE.
}

Foo() SEES THE SAME StringBuilder OBJECT WE INSTANTIATED (sb) BUT HAS AN INDEPENDENT REFERENCE TO IT. sb AND fooSB ARE SEPARATE VARIABLES THAT REFERENCE THE SAME StringBuilder OBJECT.

BECAUSE fooSB IS A COPY OF A REFERENCE, SETTING IT TO null DOESN'T MAKE sb null. (IF, HOWEVER, fooSB WAS DECLARED AND CALLED WITH THE ref MODIFIER, sb WOULD BECOME null).

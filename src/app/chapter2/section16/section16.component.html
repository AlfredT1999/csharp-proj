THE out MODIFIER

AN out ARGUMENT IS LIKE A ref ARGUMENT EXCEPT FOR THE FOLLOWING:

1. IT NEED NOT BE ASSIGNED BEFORE GOING INTO THE FUNCTION.
2. IT MUST BE ASSIGNED BEFORE IT COMES OUT OF THE FUNCTION.

THE out MODIFIER IS MOST COMMONLY USED TO GET MULTIPLE RETURN VALUES BACK FROM A METHOD:

// out modifier:
string a, b;

Split("Stevie Ray Vaughn", out a, out b);

Console.WriteLine(a); // Stevie Ray
Console.WriteLine(b); // Vaughn

void Split(string name, out string firstNames, out string lastName)
{{ squareBraketOpen }}
  int i = name.LastIndexOf(' ');
  firstNames = name.Substring(0, i); // RETURNS
  lastName = name.Substring(i + 1); // RETURNS
}

- LIKE A ref PARAMETER, AN out PARAMETER IS PASSED BY REFERENCE.

out VARIABLES AND DISCARDS

YOU CAN DECLARE VARIABLES ON THE FLY WHEN CALLING METHODS WITH out PARAMETERS. WE CAN REPLACE THE FIRST 2 LINES IN OUR PRECIDING EXAMPLE WITH THIS:

Split("Stevie Ray Vaughn", out string a, out string b);

Console.WriteLine(a); // Stevie Ray
Console.WriteLine(b); // Vaughn

void Split(string name, out string firstNames, out string lastName)
{{ squareBraketOpen }}
  int i = name.LastIndexOf(' ');
  firstNames = name.Substring(0, i); // RETURNS
  lastName = name.Substring(i + 1); // RETURNS
}

WHEN CALLING METHODS WITH MULTIPLE out PARAMETERS, SOMETIMES YOU'RE NOT INTERESTED IN RECEIVING VALUES FROM ALL THE PARAMETERS. THAT'S WHEN WE USE DISCARD "_":

Split("Stevie Ray Vaughn", out string a, out _); // Discard 2nd param
Console.WriteLine (a);

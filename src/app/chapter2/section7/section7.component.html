TYPES AND CONVERSIONS

IMPLICIT CONVERSIONS HAPPEN AUTOMATICALLY, AND EXPLICIT CONVERSIONS REQUIRE A CAST CONVERSION.

VALUE TYPES VS REFERENCE TYPES

ALL C# TYPES FALL INTO THE FOLLOWING CATEGORIES:

1. VALUE TYPES.
2. REFERENCE TYPES.
3. GENERIC TYPE PARAMETERS.
4. PONTER TYPES.

- VALUE TYPES COMPRISE MOST BUILT-IN TYPES.
- REFERENCE TYPES COMPRISE ALL CLASS, ARRAY, DELEGATE, AND INTERFACE TYPES.
- THE FUNDAMENTAL DIFFERENCE BETWEEN VALUE TYPES AND REFERENCE TYPES IS HOW THEY ARE HANDLED IN MEMORY.

VALUE TYPES

THE CONTENT OF A VALUE TYPE VARIABLE OR CONSTANT IS SIMPLY A VALUE. THE ASSIGMENT OF A VALUE-TYPE INSTANCE ALWAYS COPIES THE INSTANCE.

# REGION OF MEMORY -> &AWL_12334
# VALUE-TYPE OF THAT REGION -> 12
# REPRESENT IN C# CODE -> int variable_1 = 12;

int variable_2 = variable_1; // INTERNALLY WHAT HAPPENS IS THAT ANOTHER REGION IS USED TO STORE (COPY) variable_1;

# REGION OF MEMORY -> &AWL_56789
# VALUE-TYPE OF THAT REGION -> 12
# REPRESENT IN C# CODE -> int variable_2 = 12;

REFERENCE TYPES

A REFERENCE TYPE IS MORE COMPLEX THAN A VALUE TYPE, HAVING 2 PARTS: AN OBJECT AND THE REFERENCE TO THAT OBJECT.

REFERENCE -> POINTS TO AN OBJECT ** INSTEAD OF COPYING.

ASSIGNING THE REFERENCE-TYPE VARIABLE COPIES THE REFERENCE, NOT THE OBJECT INSTANCE. THIS ALLOWS MULTIPLE VARIABLES TO REFER TO THE SAME OBJECT.

NULL: INDICATING THAT THE REFERENCE POINTS TO NO OBJECT.

STORAGE OVERHEAD

VALUE-TYPE INSTANCES OCCUPY PRECISELY THE MEMORY REQUIRED TO STORE THEIR FIELDS.

REFERENCE TYPES REQUIRE SEPARATE ALLOCATIONS OF MEMORY FOR THE REFERENCE AND OBJECT. THE OBJECT CONSUMES AS MANY BYTES AS ITS FIELDS, PLUS ADDITIONAL ADMINISTRATIVE OVERHEAD.

FIELDS

A FIELD IS A VARIABLE THAT IS A MEMBER OF A CLASS OR STRUCT:

class Octopus
{{squareBraketOpen}}
  string name;
  public int Age = 10;
}

FIELDS ALLOW THE FOLLOWING MODIFIERS:

- STATIC MODIFIER static

- ACCESS MODIFIER public internal private protected

- INHERITANCE MODIFIER new

- UNSAFE CODE MODIFIER unsafe

- READ-ONLY MODIFIER readonly

- THREADING MODIFIER volatile

THERE ARE 2 POPULAR NAMING CONVENTIONS FOR PRIVATE FIELDS:

CAMEL-CASED AND CAMEL-CASED WITH AN UNDERSCORE (_). THE LATTER CONVENTION LETS YOU INSTANTLY DISTINGUISH PRIVATE FIELDS FROM PARAMETERS AND LOCAL VARIABLES.

THE readonly MODIFIER

THE readonly MODIFIER PREVENTS A FIELD FROM BEIGN MODIFIED AFTER CONSTRUCTION. A readonly FIELD CAN BE ASSIGNED ONLY IN ITS DECLARATION OR WITHIN THE ENCLOSING TYPE'S CONTRUCTOR.

CONSTANTS

A CONSTANT IS EVALUATED STATICALLY AT COMPILE TIME, AND THE COMPILER LITERALLY SUBSTITUTES ITS VALUE WHENEVER USED.
